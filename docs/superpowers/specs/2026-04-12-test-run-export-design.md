# Test Run Export — Design Spec

**Date:** 2026-04-12
**Status:** Approved

---

## Goal

Allow users to export a single test run as a PDF or DOCX document suitable for attaching to an RFC or change request. The export includes full detail: test results, step-by-step logs, and screenshots embedded inline.

---

## Architecture

### New module: `harness/export.py`

Owns all document generation logic. Two public functions:

```python
def export_pdf(run: dict, results: list[dict], screenshots_dir: str) -> bytes: ...
def export_docx(run: dict, results: list[dict], screenshots_dir: str) -> bytes: ...
```

Both accept the same inputs:
- `run` — dict from `db.get_run(run_id)`: `id`, `app`, `environment`, `status`, `started_at`, `finished_at`
- `results` — list of dicts from `db.get_results_for_run(run_id)`: each has `test_name`, `status`, `duration_ms`, `error_msg`, `screenshot`, `step_log` (JSON string)
- `screenshots_dir` — filesystem root for screenshots (default `"data/screenshots"`)

Returns raw bytes. The route streams these directly to the client.

**Dependencies:** `reportlab` (PDF), `python-docx` (DOCX). Both pure Python, no system dependencies.

### New route file: `web/routes/export.py`

Registers one endpoint:

```
GET /api/runs/{run_id}/export?format=pdf|docx
```

- Fetches run via `db.get_run(run_id)` → 404 if not found
- Fetches results via `db.get_results_for_run(run_id)`
- Validates `format` param — 422 if not `pdf` or `docx`
- Calls `export_pdf` or `export_docx`
- Returns `FileResponse`-style response with appropriate content-type and `Content-Disposition: attachment; filename="run-{run_id[:8]}-{app}.pdf"` (or `.docx`)

### DB change: `get_app_summary` gains `last_run_id`

Add `last_run_id` alongside the existing `last_run` timestamp in `get_app_summary`. Extended SQL:

```sql
SELECT app, id, MAX(finished_at) as last_run
FROM runs
WHERE environment = ? AND status = 'complete'
GROUP BY app
```

`last_run_id` is `None` when no completed run exists for that app.

### Route registration

`web/main.py` includes the new export router:

```python
from web.routes.export import router as export_router
app.include_router(export_router)
```

## Document Content

### Header
- Title: **Test Run Report — {app}**
- Environment, Run ID (first 8 chars), Date/Time (from `started_at`), Overall status

### Per-test section (one per test result)
- Test name, type (derived from step_log presence), status, duration
- Error message (if `error_msg` is set)
- Step log: rendered as a table with columns — Step | Status | Duration | Error
- Screenshots embedded inline (max width ~500px / equivalent):
  - Failure screenshot (`result.screenshot`) if present and file exists on disk
  - Per-step screenshots (`step.screenshot`) if present and file exists on disk
  - Missing files are skipped silently — no exception raised

### Footer
- "Generated by Web Testing Harness on {datetime}"

## UI Entry Points

### Detail page (`detail.html`)

Two small buttons added to the `.run-meta` bar, next to "Re-run All":

```html
<a href="/api/runs/{{ selected_run.id }}/export?format=pdf" class="btn btn-sm">Export PDF</a>
<a href="/api/runs/{{ selected_run.id }}/export?format=docx" class="btn btn-sm">Export DOCX</a>
```

### Dashboard (`dashboard.html`)

Two small export links in each app row's action column, rendered only when `last_run_id` is non-null:

```html
{% if row.last_run_id %}
<a href="/api/runs/{{ row.last_run_id }}/export?format=pdf" class="btn btn-sm">PDF</a>
<a href="/api/runs/{{ row.last_run_id }}/export?format=docx" class="btn btn-sm">DOCX</a>
{% endif %}
```

## Error Handling

| Condition | Response |
|-----------|----------|
| `run_id` not found | 404 JSON |
| `format` not `pdf` or `docx` | 422 JSON |
| Screenshot file missing on disk | Skip silently, continue export |
| Export library raises | 500 JSON with error detail |

## Testing

### `tests/test_export.py`

**Unit tests (`export_pdf`, `export_docx`):**
- Returns non-empty bytes for a run with no screenshots
- PDF bytes start with `%PDF`
- DOCX bytes start with PK zip magic bytes (`b'PK'`)
- Run with a failure screenshot — file found → embedded; file missing → skipped without error
- Run with step screenshots — embedded when present, skipped when missing

**Route tests:**
- `GET /api/runs/{run_id}/export?format=pdf` → 200, `Content-Type: application/pdf`
- `GET /api/runs/{run_id}/export?format=docx` → 200, `Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document`
- Unknown run_id → 404
- `format=xml` → 422

### `tests/test_models_and_db.py`

- `get_app_summary` includes `last_run_id` when a completed run exists
- `get_app_summary` returns `last_run_id: None` when no completed run exists
